# Contributing

Thank you for your interest in contributing! This is an open-source project and contributions of all kinds are welcome — bug fixes, improvements to templates, documentation updates, or new features.

## Getting Started

1. **Fork** this repository
2. **Clone** your fork locally
3. Create a new branch: `git checkout -b your-feature-name`
4. Make your changes
5. **Commit** with a clear message: `git commit -m "feat: describe your change"`
6. **Push** to your fork: `git push origin your-feature-name`
7. Open a **Pull Request** against the `main` branch

## What You Can Contribute

- **Bug fixes** — incorrect rendering, broken links, template issues
- **New templates** — additional resume or profile layout variants
- **Improvements** — better styling, accessibility enhancements, print optimizations
- **Documentation** — clearer instructions, additional examples
- **Data schema** — suggestions to improve or extend the JSON structure

## Guidelines

- Keep changes focused — one fix or feature per pull request
- Follow the existing code style and file structure
- Test your changes in a browser before submitting
- Update documentation if your change affects usage
- Be descriptive in your PR — explain what you changed and why

## Issues

Found a bug or have a suggestion? Open an issue with a clear title and enough detail to reproduce or understand the request. Check existing issues first to avoid duplicates.

## Code of Conduct

All contributors are expected to follow the [Code of Conduct](CODE_OF_CONDUCT.md).

We appreciate every contribution, no matter how small. Thank you for helping improve this project!
